影响 Agent 的理论基础（1）

ReAct

论文：ReAct: Synergizing Reasoning and Acting in Language Models

作者：Shunyu Yao 等

Source：arXiv:2210.03629

核心思想：把“思考”和“执行”放进同一条链路，让模型不再停留在文本回答，而是通过行动结果不断修正

它被很多后续 Agent 系统当作基线思路：先做、再看、再改，而不是一次性输出最终答案

应用与影响

  • 在交互任务（ALFWorld / WebShop）中显著提升成功率，验证了闭环策略有效性（论文实验）
  • 推动了后续 Agent 框架将“思考链 + 工具调用 + 观测反馈”作为标准模式
  • 工程上被广泛用作“先推理再执行，再根据结果修正”的模板

Act

影响 Agent 的理论基础（2）

Toolformer

论文：Toolformer: Language Models Can Teach Themselves to Use Tools

作者：Timo Schick 等（Meta AI, 2023）

Source：arXiv:2302.04761

论文结论：通过少量 API 示例进行自监督学习，模型可学会何时调用、如何调用并融合工具结果

示例
候选调用
筛选
嵌入推理

应用层意义：推动“模型自主工具调用”成为主流能力，影响后续 tool calling 与 agent API 设计

Reflexion

论文：Reflexion: Language Agents with Verbal Reinforcement Learning

作者：Noah Shinn 等（Princeton / DeepMind）

Source：arXiv:2303.11366

通俗理解：让 Agent 每次做完先“复盘一句话”，把经验写进记忆里，下一次少走弯路

执行
反思
写入记忆
下轮复用

应用层意义：推动“语言反思 + 记忆复用”成为无重训迭代的重要路线

Skills 与 MCP

Skills

把高频任务封装成可复用模块，减少每次从零开始

  • 搜索
  • 代码生成
  • 文件处理

本次 PPT 就是通过 frontend slides skill 生成并迭代完成的

MCP（Model Context Protocol）

MCP 是连接模型与外部系统的标准协议，可理解为 AI 的 USB-C 接口

它把工具接入方式标准化，降低系统集成成本，提升 Agent 可维护性

Toolformer 解决“会不会用工具”，MCP 解决“工具怎么接入”
